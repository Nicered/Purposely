Purposely

Purpose-Driven Development Framework

What is Purposely?

Purposely is a CLI framework that helps development teams maintain project purpose throughout the entire development lifecycle. It integrates seamlessly with Claude Code to provide structured documentation and workflow management.

Quick Start

Installation

# Using uvx (recommended)
uvx --from git+https://github.com/nicered/purposely purposely init

# Or install globally
pip install git+https://github.com/nicered/purposely
purposely init

Initialize Your Project

# Start a new project
purposely init

# Add to existing project
purposely init --existing

Development Workflow

  1. Define Purpose - Use /purposely-init to establish project goals
  2. Create Phase - Use /purposely-phase to plan a development phase
  3. Research - Use /purposely-research to investigate technical decisions
  4. Design - Use /purposely-design to create system architecture
  5. Plan - Use /purposely-plan to detail implementation tasks
  6. Implement - Use /purposely-implement for guided development

Available Commands

CLI Commands

purposely init              # Initialize new project
purposely init --existing   # Add to existing project
purposely create spec       # Create phase specification
purposely create research-overview  # Create research overview
purposely create design-overview    # Create design overview
purposely upgrade           # Upgrade templates
purposely upgrade --force   # Force upgrade

Claude Code Slash Commands

/purposely-init         # Initialize project purpose
/purposely-phase        # Start new development phase
/purposely-research     # Conduct technical research
/purposely-design       # Create system design
/purposely-plan         # Plan implementation
/purposely-implement    # Execute implementation

Why Purpose-Driven Development?

  • Maintain Focus - Never lose sight of project goals
  • Better Decisions - Every choice is evaluated against project purpose
  • Clear Documentation - Structured docs that tell the whole story
  • Team Alignment - Everyone understands the "why" behind the code
  • Knowledge Preservation - Capture decisions and rationale as you build
